# 用户测试指南

**版本**: 1.0
**更新时间**: 2026-02-06
**目标**: 验证 FeishuMind MVP 的核心功能

---

## 📋 测试概览

### 测试目标

验证以下核心功能是否满足用户需求:

1. ✅ 基础对话交互
2. ✅ 事件提醒功能
3. ✅ 情绪分析和韧性辅导
4. ✅ 记忆存储和检索
5. ✅ GitHub 热门推送

### 测试人群

- **内部测试**: 开发团队成员
- **Alpha 测试**: 5-10 个飞书用户
- **Beta 测试**: 50-100 个飞书用户

---

## 🎯 测试场景

### 场景 1: 日常对话 (必测)

**目标**: 验证基础对话能力

**步骤**:
1. 在飞书中向 FeishuMind 发送消息: "你好"
2. 验证是否收到欢迎回复
3. 发送: "你能帮我做什么？"
4. 验证功能介绍是否清晰

**预期结果**:
- ✅ 响应时间 < 3秒
- ✅ 回复内容友好、专业
- ✅ 包含功能介绍

**问题记录**:
- 响应时间: ____ 秒
- 回复质量: ⭐⭐⭐⭐⭐ (1-5星)
- 问题描述: ________________

---

### 场景 2: 创建事件提醒 (必测)

**目标**: 验证事件提醒功能

**步骤**:
1. 发送: "提醒我明天下午3点开会"
2. 验证是否正确解析时间
3. 检查飞书日历是否创建事件
4. 验证是否设置了提醒

**预期结果**:
- ✅ 时间解析准确 (明天下午3点)
- ✅ 飞书日历成功创建事件
- ✅ 提醒设置正确 (提前15分钟/1小时/1天)

**问题记录**:
- 时间解析: ✅ / ❌
- 日历创建: ✅ / ❌
- 提醒设置: ✅ / ❌
- 问题描述: ________________

### 场景 3: 情绪分析 (必测)

**目标**: 验证情绪分析和韧性辅导

**步骤**:
1. 发送: "这周项目压力很大，经常加班"
2. 验证是否识别出压力情绪
3. 检查是否提供合理建议
4. 验证建议的可执行性

**预期结果**:
- ✅ 识别出"压力"情绪
- ✅ 提供非医疗建议
- ✅ 建议具体可执行

**问题记录**:
- 情绪识别准确度: ⭐⭐⭐⭐⭐
- 建议质量: ⭐⭐⭐⭐⭐
- 是否有医疗建议: 是 / 否 (应该是否)
- 问题描述: ________________

### 场景 4: 记忆功能 (必测)

**目标**: 验证记忆存储和检索

**步骤**:
1. 发送: "我喜欢在早上处理高难度任务"
2. 等待 10 秒
3. 发送: "我习惯什么时候处理高难度任务？"
4. 验证是否从记忆中检索到正确答案

**预期结果**:
- ✅ 成功存储偏好
- ✅ 准确检索到"早上"
- ✅ 响应时间 < 3秒

### 场景 5: GitHub 热门推送 (可选)

**目标**: 验证 GitHub Trending 功能

**前提条件**:
- 已配置 GitHub Token
- 已配置定时任务

**步骤**:
1. 等待定时任务触发 (或手动触发)
2. 检查是否收到飞书卡片
3. 验证卡片内容格式
4. 点击链接验证有效性

**预期结果**:
- ✅ 成功收到推送
- ✅ 卡片格式正确
- ✅ 链接有效

**问题记录**:
- 推送成功: ✅ / ❌
- 卡片格式: ⭐⭐⭐⭐⭐
- 链接有效: ✅ / ❌
- 问题描述: ________________

---

### 场景 6: 多轮对话 (必测)

**目标**: 验证上下文理解能力

**步骤**:
1. 第一轮: "我想安排一个会议"
2. 第二轮: "明天下午3点"
3. 第三轮: "关于项目进度汇报"
4. 验证是否正确理解上下文

**预期结果**:
- ✅ 理解"会议"是主题
- ✅ 理解"明天下午3点"是时间
- ✅ 理解"项目进度汇报"是内容
- ✅ 创建完整的事件

**问题记录**:
- 上下文理解: ⭐⭐⭐⭐⭐
- 事件完整性: ✅ / ❌
